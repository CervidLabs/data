# Security Policy

## Supported Versions

Currently, only the latest stable version of **Cervid** receives security updates. We recommend always staying up to date with the latest release to ensure maximum protection.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of Cervid very seriously. If you believe you have found a security vulnerability, please do **not** open a public issue. Instead, follow this protocol:

### How to report
Please report vulnerabilities privately by sending an email to **[support@cervid.io]**. 

### What to include
To help us prioritize and resolve the issue faster, please include:
1. A brief description of the vulnerability.
2. Steps to reproduce the issue (proof of concept).
3. Potential impact if exploited.

### Our Commitment
* **Response Time:** You can expect an initial acknowledgement of your report within **48 hours**.
* **Updates:** We will provide status updates at least once a week until the vulnerability is resolved.
* **Disclosure:** Once the fix is deployed, we will coordinate with you to provide appropriate credit and decide on a public disclosure date.

Thank you for helping us keep the Cervid engine secure!
